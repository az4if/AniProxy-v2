Features

Everything your anime app needs

AniProxy sits between your frontend and Anilist's GraphQL API — handling CORS, auth headers, and errors so you don't have to.

🌐

CORS — fully resolved

Browser security blocks direct calls to external APIs. AniProxy runs on your own domain, returning permissive CORS headers so any web client can query Anilist without a single browser warning or preflight error.

🔐

Auth header passthrough

Forwards your Authorization header verbatim so private user data and mutations work exactly as they do against Anilist directly.

⚡

Serverless — edge-fast

Deployed as a Vercel serverless function. Cold-start is under 100 ms; warm requests typically sit below 50 ms end-to-end.

🔄

Structured error responses

Surfaces Anilist GraphQL errors, network failures, and malformed payloads as consistent JSON — making client-side debugging straightforward.

🎯

Drop-in replacement

The proxy is request/response-compatible with graphql.anilist.co. Change one URL in your existing code and you're done — no schema changes, no new client libraries.

🚀

One-click deploy

Clones the repo and deploys to Vercel in about 30 seconds. No env vars, no build config, nothing to set up.

How it works

Simple by design

Three steps from idea to live proxy endpoint.

Step 01

Deploy

Click "Deploy to Vercel". The repo is cloned and a serverless function is provisioned automatically — no local setup or env vars needed.

→
Step 02

Point your app

Swap graphql.anilist.co for your Vercel URL + /api/graphql in your client code. That's the only change.

→
Step 03

Ship

Every GraphQL query and mutation routes through the proxy. CORS is handled, errors are structured, and Anilist's full API surface is available.

Quickstart

Start querying in seconds

Copy, paste, replace the URL — and you're querying Anilist from any browser.

query.js
const query = ` query ($id: Int) { Media(id: $id, type: ANIME) { id title { romaji english } description episodes averageScore } } `; const response = await fetch('https://your-deploy.vercel.app/api/graphql', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ query, variables: { id: 15125 } }), }); const { data } = await response.json(); console.log(data.Media.title.romaji); // "Fullmetal Alchemist: Brotherhood"
terminal
curl -X POST \ https://your-deploy.vercel.app/api/graphql \ -H 'Content-Type: application/json' \ -d '{ "query": "query { Page(perPage: 5) { media(type: ANIME, sort: POPULARITY_DESC) { title { romaji } averageScore } } }" }'
query.py
import requests query = """ query ($id: Int) { Media(id: $id, type: ANIME) { id title { romaji english } averageScore } } """ response = requests.post( 'https://your-deploy.vercel.app/api/graphql', json={'query': query, 'variables': {'id': 15125}}, ) data = response.json()['data']['Media'] print(data['title']['romaji']) # Fullmetal Alchemist: Brotherhood
